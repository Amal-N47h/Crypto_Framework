# Secure IoMT Data Storage with Integrity Verification

## 📌 Description

This module implements persistent storage for encrypted IoMT sensor records using SQLite. It stores encrypted payloads generated by the hybrid packet structure in `crypto_AmalNath/hybrid_encrypt.py`, along with the device ID, timestamp, optional IV, ephemeral public key, and SHA-256 hash.

The system verifies stored ciphertext integrity during retrieval by recomputing its SHA-256 hash and comparing it with the stored value.

---

## 🎯 Objectives

* Securely store encrypted IoMT data
* Support AmalNath's hybrid packet format (`encrypted_data` + `ephemeral_public_key`)
* Attach integrity proof (SHA-256 hash)
* Detect tampering during data retrieval
* Optimize queries using indexing

---

## 🗄️ Database Design

### Table: `secure_sensor_data`

| Field          | Type    | Description                    |
| -------------- | ------- | ------------------------------ |
| id             | INTEGER | Primary Key                    |
| device_id      | TEXT    | Unique device identifier       |
| timestamp      | INTEGER | Record creation time           |
| iv             | BLOB    | AES initialization vector, or empty bytes when not supplied by the packet |
| encrypted_data | BLOB    | Ciphertext                     |
| ephemeral_public_key | BLOB | Sender ephemeral ECC public key from the hybrid packet |
| hash           | TEXT    | SHA-256 hash of encrypted data |

---

## ⚙️ Indexing Strategy

Indexes are added to:

* `device_id` → fast device-based queries
* `timestamp` → fast time-range queries

---

## 🔐 Integrity Mechanism

* SHA-256 hash is generated from encrypted data before storage
* On retrieval, hash is recomputed
* If mismatch occurs → record is flagged as **tampered**
* The hash only proves storage integrity; authenticity must come from a keyed MAC or signature in another module

---

## 🔄 Workflow

1. Hybrid encryption module builds a packet using `build_packet(encrypted_data, ephemeral_public_key)`
2. `db.py` receives the packet through `insert_hybrid_packet(device_id, packet, iv=b"")`
3. `db.py` extracts:

   * `encrypted_data`
   * `ephemeral_public_key`
   * optional `iv`

4. SHA-256 hash is generated from `encrypted_data`
5. Data + metadata are stored in SQLite
6. On retrieval through `read_and_verify(record_id)`:

   * Recompute hash
   * Compare with stored value
   * Return `None` if tampering is detected
   * Rebuild and return the hybrid packet fields with database metadata if valid

---

## ⚠️ Security Notes

* Hashing encrypted data ensures detection of storage-level tampering
* SHA-256 without a secret key does not prove authenticity
* IV should be stored when the encryption mode requires it for decryption
* Current integration depends on AmalNath's packet builder, but actual hybrid encryption/decryption is still implemented separately
* Existing database files are migrated by adding `ephemeral_public_key` if the column is missing
